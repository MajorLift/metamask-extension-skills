# Installing metamask-extension-skills

Three paths. Pick whichever fits your environment. All three install the same shell tools; the repo path additionally installs stable skills into your project.

## Distribution model

Follows the Agent Skills standard. One canonical location per skill, pointer shims for the other adapter formats — no content duplication.

```
<repo>/.agents/skills/<slug>/SKILL.md    ← canonical
<repo>/.claude/skills/<slug>/SKILL.md    ← symlink → canonical
<repo>/.cursor/rules/<slug>.mdc          ← symlink → canonical
```

All generated directories are meant to be gitignored (see below). The skills repo is the source of truth; consuming repos just mirror and regenerate.

## 1. Consuming repo (recommended for engineers)

Add to `package.json`:

```json
{
  "scripts": {
    "skills:sync": "bash <(curl -fsSL https://raw.githubusercontent.com/MajorLift/metamask-extension-skills/main/tools/sync.sh) --target ."
  }
}
```

Then:

```bash
yarn skills:sync
```

What it does:

- Clones `MajorLift/metamask-extension-skills` at `main` (or a pinned ref — see below)
- Writes canonical stable skills to `.agents/skills/<slug>/SKILL.md`
- Generates symlink shims at `.claude/skills/<slug>/SKILL.md` and `.cursor/rules/<slug>.mdc`
- Copies `/remember`, `/patch`, `/backfill` command specs to `.claude/commands/` and `.cursor/rules/`
- Copies shell fallbacks to `.skills/`
- Writes `.skills/VERSION` with synced ref + sha + timestamp

Re-run weekly (or any time you want the latest skills). `/remember` warns if it's been >3 days since last sync.

## Pinning a ref (supply-chain safety)

By default sync tracks `main`. Pin to a tag, branch, or SHA to freeze what lands in your repo:

```bash
yarn skills:sync -- --ref v2026.04.20
bash tools/sync.sh --target . --ref 4a7c1f2
```

The resolved SHA is stamped into `.skills/VERSION`. Teams that require reproducible syncs should pin to a tag and bump intentionally.

## 2. curl | bash (no repo or ad-hoc)

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/MajorLift/metamask-extension-skills/main/tools/sync.sh) --target .
```

Same behavior as yarn path, no package.json change.

For user-level install (commands available globally, no repo writes):

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/MajorLift/metamask-extension-skills/main/tools/sync.sh) --target . --user
```

Writes to `~/.claude/commands/`, `~/.cursor/rules/`, `~/.skills/`.

## Tools-only install (scripts without skills)

If you only want the shell tools and no skill file drops:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/MajorLift/metamask-extension-skills/main/tools/sync.sh) --target . --tools-only
```

## Including draft skills

By default only `maturity: stable` skills are installed. To preview in-review skills:

```bash
bash tools/install.sh --target . --include-maturity stable,draft
```

Skills marked `experimental` are never synced to consuming repos.

## After install

```bash
# Claude Code / Cursor slash commands
/remember "prefer pnpm over npm in this repo"
/patch pnpm-over-npm "exception: use npm inside docker builds"
/backfill

# Shell fallback (any terminal)
./.skills/remember.sh "prefer pnpm over npm in this repo"
./.skills/patch.sh pnpm-over-npm "exception: use npm inside docker builds"
```

Requires `gh` CLI authenticated with write access to `MajorLift/metamask-extension-skills`.

## Uninstalling

```bash
rm -rf .claude/skills .cursor/rules .agents/skills .skills .claude/commands/{remember,patch,backfill}.md
```

User-level:

```bash
rm -rf ~/.skills ~/.claude/commands/{remember,patch,backfill}.md ~/.cursor/rules/{remember,patch,backfill}.mdc
```

## Windows / no-symlink filesystems

Pass `--no-symlinks` to copy shim files instead of symlinking. Skills will be duplicated on disk but the Agent Skills layout is preserved.

```bash
bash tools/sync.sh --target . --no-symlinks
```

## Gitignore

Add these to your consuming repo's `.gitignore` (all four are generated by every sync):

```
/.agents/skills/
/.claude/skills/
/.cursor/rules/
/.skills/
```

Never hand-edit generated files — edit source in `metamask-extension-skills` and re-sync.

## Checking your sync state

```bash
cat .skills/VERSION
# synced_ref=main
# synced_sha=abc1234f5e6d7c8b9a0
# synced_at=2026-04-20T15:32:11Z
# source=https://github.com/MajorLift/metamask-extension-skills.git
```

## Dashboard

Inside the skills repo:

```bash
./scripts/skills-count.sh
./scripts/skills-count.sh --json  # for CI
```

Shows total / by origin / by maturity / by domain / captures by author / patches by author / latest 5 captures / latest 5 patches.
